# 🎯 从这里开始！

欢迎使用植物病害检测系统！如果您是第一次部署，请按照以下顺序阅读文档。

## 📋 推荐阅读顺序

### 第一步：了解系统
1. 阅读 [README.md](./README.md) - 了解系统架构和功能
2. 阅读 [DEPLOYMENT_SUMMARY.md](./DEPLOYMENT_SUMMARY.md) - 了解系统组成

### 第二步：准备部署
1. **必读**：[完整部署指南（小白版）](./COMPLETE_DEPLOYMENT_GUIDE.md)
   - 这是最详细的指南，适合完全没有经验的新手
   - 包含服务器购买、配置、部署的每一步
   - 预计阅读时间：30分钟
   - 预计部署时间：1-2小时

2. 准备 [部署检查清单](./DEPLOYMENT_CHECKLIST.md)
   - 打印或保存此清单
   - 部署时逐项检查

### 第三步：开始部署
按照 [完整部署指南](./COMPLETE_DEPLOYMENT_GUIDE.md) 的步骤执行：

1. ✅ 购买服务器
2. ✅ 连接服务器
3. ✅ 上传代码
4. ✅ 运行部署脚本
5. ✅ 配置环境变量
6. ✅ 上传模型
7. ✅ 测试验证

### 第四步：遇到问题？
如果遇到问题，查看：
- [故障排除指南](./TROUBLESHOOTING_GUIDE.md) - 常见问题解决方案
- 查看服务日志：`docker-compose logs 服务名称`

### 第五步：ESP32设备（可选）
如果使用ESP32边缘设备：
- 参考 [完整部署指南](./COMPLETE_DEPLOYMENT_GUIDE.md) 中的ESP32章节
- 查看 [edge-devices/esp32/README.md](./edge-devices/esp32/README.md)

## 🎓 学习路径

### 完全新手
1. 阅读完整部署指南（30分钟）
2. 准备服务器和代码（30分钟）
3. 执行部署步骤（1-2小时）
4. 测试验证（30分钟）
5. 遇到问题查看故障排除指南

### 有Docker经验
1. 阅读快速开始（5分钟）
2. 运行一键部署脚本（10分钟）
3. 配置环境变量（5分钟）
4. 上传模型（10分钟）
5. 测试验证（10分钟）

### 有完整经验
1. 直接运行 `scripts/one_click_deploy.sh`
2. 配置 `.env` 文件
3. 上传模型文件
4. 完成！

## 💡 重要提示

1. **不要跳过步骤**：即使是看起来简单的步骤，也可能影响后续部署
2. **记录信息**：部署过程中记录所有密码、IP地址、配置信息
3. **查看日志**：遇到问题时，首先查看日志文件
4. **备份数据**：重要操作前记得备份
